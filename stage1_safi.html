Thinking Lenses

As a programmer, you have to develop specific thinking lenses that you will alternate between as you are creating your code. Each len has its advantages for each stage of the coding process. The lenses are as follow:

1. Procedural Thinking

Like saying, (1) ..., (2) ..., (3) ..., and so on.

2. Abstract Thinking

Like saying, 1 + 1 = 2

3. Systematic Debugging

Like saying, The problem is ..., and the possible causes coule be ...

4. Technological Empathy

This is the lens where the user get to understand how the computer sees things.

5. Systems Thinking

This is the most important lens. This lens helps breaking down big ideas into component pieces.

HTML & CSS

HTML stands for Hypertext Markup Language while CSS stands for Cascading Style Sheet. HTML is primarily used for structuring the document code as well as inputing the content. On the other hand, the CSS is useful for how the content is being presented on the website. Note that styling can be done within the HTML code. However, it is preferred to coding style in a seperate file ".css" then link it to ".html".

Typical HTML Document Structure

<! Doctype HTML> <html> <head> <title>TheTitle</title> </head> <body> The Content along with its elements and tags </body> </html>

HTML Elements Attributes

Some elements requires attribues to be added within a tag. For example, an images needs to be directed to its source and a link (URL) should be referenced to its URL link. The attribute is included in the opening tag in this form:
< a href="URL" > Content . In other words, an element with an attribute with have the following components: Within the opening tag (an attribute name, a value), and content followed by a closing tag where appropriate.

Understanding Document Code Structrue ( Div and Span )

Before getting into what a structure means here, it is important to understand the relationships between the lines of codes. e.g. a child, a parent, or a sibling. Structuring a document with indentation within the code does not tell HTML anything since the whitespace is ignored. Thus, the tags <div></div> & <span></span> are widely used for the purpose of structuring the document.
Now, the div tag also plays an important role when linking CSS with an HTML file. The div tag is used as a selector when defining a class for it. For example, in my HTML code I would write an attribute "class" in my div tag and add the value (string) while will be referenced in the CSS code in order to style it.

HTML "Classes" and Avoiding Repetition

If you wondered why do we use CSS for styling while it is possible to do so in HTML. Well, there are several benefits for doing so.
  1. Most importantly is the idea of avoiding repetition within the code.
  2. Focusig on the content within the HTML code helps making the presentation better later on
  3. Linking multiple .css files to one HTML comes in handy when working on a full website. This also helps to simply apply different .css for the same .html depending on which device (i.e. laptop, phone, or tablet) the web page is being viewed from.

Terms

HTML Element:
Elements types are indicated by HTML tags. Tags are written like this < > and a letter in between indicating what tag it is. Each element has an opening tag and a closing tag; Except some element like the ones for an image . The element without closing tags are referred to as an element with a void tag.
Elements are divided in two main categories: Inline and Block.
  • Inline Elements: a, span, br, img, strong
  • Block elements: div, p, form

Web Page:
A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.

World Wide Web (WWW)
The World Wide Web is a collection of HTML documents which are connected through this web.

Tags
Keep in mind that if a tag is not closed, everything after the tag is activated by that tag. Yes, computers are stupid. They do exactly what YOU tell them to do. Thus, be careful about how you communicate with a computer and understand how it understands you :).
  • <i> & <em> : Both make the content italics. However, emphasis <em> is becoming more pupolar for advanced coding reasons
  • <b> & <strong> : bold vs. strong, they both bold the word.
